# Руководство по настройке Docker

Это руководство содержит инструкции для запуска приложения pyesb с использованием Docker.

## Быстрый старт

### Сборка и запуск
```bash
# Сборка Docker-образа
docker build -t pyesb .

# Запуск контейнера
docker run -d --name pyesb-container -p 9090:9090 pyesb

# Просмотр логов
docker logs pyesb-container
```

### Проверка работоспособности
```bash
# Тестирование health-эндпоинта
curl http://localhost:9090/health

# Должно вернуть: {"status":"ok"}
```

## Использование Docker Compose (рекомендуется)

Для упрощенного управления используйте предоставленный файл docker-compose.yml:

```bash
# Запуск приложения
docker-compose up -d

# Проверка статуса
docker-compose ps

# Просмотр логов
docker-compose logs -f app

# Остановка приложения
docker-compose down
```

## Опции конфигурации

### Переменные окружения

Вы можете переопределить параметры по умолчанию, используя переменные окружения:

| Переменная | По умолчанию | Описание |
|----------|---------|-------------|
| `PORT` | 9090 | Порт HTTP для прослушивания |
| `HOST` | 0.0.0.0 | Адрес хоста для привязки |
| `LOG_LEVEL` | info | Уровень логирования (debug, info, warning, error) |

Пример:
```bash
docker run -d --name pyesb-container \
  -p 8080:8080 \
  -e PORT=8080 \
  -e LOG_LEVEL=debug \
  pyesb
```

### Постоянное хранение ключей

По умолчанию, RSA-ключи хранятся внутри контейнера. Чтобы сделать их постоянными:

```bash
# Создание локальной директории для ключей
mkdir -p ./keys

# Запуск с монтированием тома
docker run -d --name pyesb-container \
  -p 9090:9090 \
  -v ./keys:/app/keys \
  pyesb
```

## Режим разработки

Для разработки вы можете смонтировать директорию приложения, чтобы включить перезагрузку кода на лету:

```bash
# Сначала соберите образ (для получения зависимостей)
docker build -t pyesb .

# Запуск со смонтированным исходным кодом
docker run -d --name pyesb-dev \
  -p 9090:9090 \
  -v ./app:/app/app \
  -v ./keys:/app/keys \
  -e LOG_LEVEL=debug \
  pyesb
```

**Примечание**: Автоперезагрузка FastAPI работает не очень хорошо в Docker. Для разработки запускайте приложение напрямую:
```bash
uv run fastapi app.main:app --reload
```
